我在2024年很简单的看过一些论文，大概的知道了transformer架构是个什么东西，粗略的认为，就是三个矩阵，其实有点像欧拉函数的三视图的样子，但要我说每一个矩阵具体负责什么，哈哈我确实不知道，但我认为，一个模型如果是基于纯数据的，哪怕它是所谓的多模态，哪怕它叠加了meta-learning，其实说到底，也只是在一个，使用了穷举法的集合里，把那些出现符合规则的，密度比较高的点，挑选了出来，抛弃了度量，用概率去表达点与点之间的距离；

这其实在技术上是对的，但是在数学上，是不具备逻辑的；
前些天我突然寻思，似乎，哲学是人类对世界规律的总结，那么逻辑学，就是语言层面对规律的表达；数学，就是逻辑学的量化表达；物理学，就是数学的技术表达；计算机，就是把逻辑学用基础计算机语言，也就是物理层面表达的学科；

但是很多问题都没解决；

在我暑假期间，我预习了大三的度量空间，用的是introduction to metric and topological spaces这本书，在我学习微积分的时候，我就问我的老师，为什么梯度的方向一定是朝着固定的更高的方向去的，老师说：这个问题是以后的知识。就不再回答我了；今天我想，如果在最优化的角度上，大概是在一个由微分方程构成的空间里，找到那个所谓的平衡点吧。

那么在我学习度量空间的时候，我也有一些疑问，
比如：
为什么度量空间的性质围绕着connectedness, completeness, compactness三者，为什么它们可以被定义为众多性质的核心？一下子，就把问题引到了拓扑学和集合论。
对于direct image f(X)，为什么f(the intersection of X) is the subset of the intersection of f(X), 我换了一个思路去理解：
  第一、是系统论的想法，这是我在'系统之美'这本书里看到的，系统往往具备各组成成分之和以外的性质，所以这里子集的方向是确定的；
  第二、是信息的角度，如果把集合看作是一种信息的集合，那么映射，就是一种通道，而inverse image代表的，就是一种无损的信息传输，而一旦这么理解，对于其余的性质，也就无非是这个结论在交并差集的应用了；
当然，还有康托集，书上说希尔伯特说我们的数学是建立在康托的基础上的，但是问题在于，数学定理的合理性，只能证明它符合当下已有的数学框架，并不能证明它就真的正确；

这么多的想法，还有最后我对度量空间的总结，让我觉得，我不知道的东西，实在太多了...可是单单是复杂系统的新三论，旧三论，又新三论恐怕都要我用一生的时间去学习，更何况要研究透彻，还需要结合多种学科...

我不知道怎么办...

说到模型上也是如此：
如果考虑对一个神经网络叠加无数个meta-layer，到最后的效果一定是微乎其微的，那是否有一种'解密'的方法，把这无数个meta-layer，总结为一种范式，而这种简单的范式，可以像分形那样，不断地发散，但最后的结果却是收敛成形态的；

如何把非量化的语言，让模型可以转化成真正意义的逻辑语言，量化语言，这其实就是一种建模思想；

如果给模型设置一些最底层的公理让它学习，那么数学的公理，哲学的公理都有可能会限制它，人为什么有智能，就在于人，可以推翻自己的想法，人可以想象，可以假设场景和后果；这些东西，我不认为是生物层面的，所以我也不认为机器做不到，我反而认为，机器就应该做到；

对于一个未经标注的数据集，如果用概率去学习，那现在的AI就完蛋了，标注是什么？降低噪声？不是的，AI不懂得评判批判，所以要人来去做这件事；
